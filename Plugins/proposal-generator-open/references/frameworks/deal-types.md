# Deal Types Reference

## Overview

Select the deal type that best matches your opportunity. This selection determines:
- Template structure and sections
- Page length and depth
- Tone and formality
- Compliance requirements

---

## GOVERNMENT

### When to Use
- Public sector RFPs
- Government agencies (federal, state, local)
- Heavily regulated industries
- Formal procurement processes
- When compliance documentation is required

### Template: `government-style.md`

### Characteristics
| Attribute | Specification |
|-----------|---------------|
| Page Range | 20-30 pages |
| Tone | Formal, compliance-focused |
| Structure | Section-numbered, matches RFP |
| Emphasis | Audit trail, security, past performance |

### Required Sections
1. Executive Summary
2. Understanding of Requirements
3. Technical Approach
4. Management Approach
5. Past Performance
6. Staffing Plan
7. Price Proposal
8. Data Security & Compliance
9. Appendices (certifications, attestations)

### Key Compliance Elements
- EEOC, OFCCP (if workforce-related)
- NIST Cybersecurity Framework
- FedRAMP (if cloud-based)
- Section numbering matches RFP exactly
- Compliance matrix required
- Certifications appendix

---

## COMMERCIAL

### When to Use
- Private sector companies
- Business-to-business sales
- ROI-driven decision makers
- Flexible procurement processes
- When speed matters more than formality

### Template: `commercial-style.md`

### Characteristics
| Attribute | Specification |
|-----------|---------------|
| Page Range | 12-15 pages |
| Tone | Professional, outcome-focused |
| Structure | Narrative flow, less formal |
| Emphasis | ROI, time savings, business impact |

### Required Sections
1. Executive Summary
2. The Challenge You're Facing
3. Our Approach
4. Your Investment
5. Why [Your Company]
6. Next Steps

### Key Elements
- Lead with ROI and business case
- Case studies with quantified results
- Clear, simple pricing
- Strong call to action
- Flexible structure based on client preferences

---

## PARTNERSHIP

### When to Use
- Joint ventures
- Strategic alliances
- Collaborative projects
- Universities and nonprofits
- When shared risk/reward applies

### Template: `partnership-style.md`

### Characteristics
| Attribute | Specification |
|-----------|---------------|
| Page Range | 10-15 pages |
| Tone | Collaborative, mutual benefit |
| Structure | Narrative, story-driven |
| Emphasis | Shared mission, impact, sustainability |

### Required Sections
1. Executive Summary
2. Shared Vision
3. Each Party's Contribution
4. Expected Outcomes
5. Governance Structure
6. Risk and Reward Sharing
7. Timeline and Milestones
8. Next Steps

### Key Elements
- Emphasize mutual benefit
- Long-term vision orientation
- Joint success metrics
- Shared governance model
- Flexible terms

---

## SIMPLE / SOW

### When to Use
- Straightforward project scopes
- Quick turnaround needed
- Existing client relationships
- Clear deliverables and timeline
- When brevity is valued

### Template: `simple-sow.md`

### Characteristics
| Attribute | Specification |
|-----------|---------------|
| Page Range | 3-8 pages |
| Tone | Concise, direct |
| Structure | Minimal, action-oriented |
| Emphasis | Scope, deliverables, timeline, price |

### Required Sections
1. Scope of Work
2. Deliverables
3. Timeline
4. Investment
5. Assumptions
6. Acceptance

### Key Elements
- Get to the point quickly
- Clear deliverables table
- Simple timeline
- Straightforward pricing
- Minimal boilerplate

## Selection Decision Tree

```
START
  │
  ├── Is this a government/public sector opportunity?
  │   ├── YES → Is there a formal RFP with required sections?
  │   │         ├── YES → GOVERNMENT
  │   │         └── NO → Does it require compliance documentation?
  │   │                   ├── YES → GOVERNMENT
  │   │                   └── NO → COMMERCIAL
  │   │
  │   └── NO → Is this a partnership or collaboration?
  │             ├── YES → Is there shared risk/reward?
  │             │         ├── YES → PARTNERSHIP
  │             │         └── NO → COMMERCIAL
  │             │
  │             └── NO → Is the scope straightforward?
  │                       ├── YES → Is this an existing client?
  │                       │         ├── YES → SIMPLE/SOW
  │                       │         └── NO → COMMERCIAL
  │                       └── NO → COMMERCIAL
```

## Mixing Deal Types

Sometimes you need elements from multiple types:

| Situation | Primary | Add From |
|-----------|---------|----------|
| Government client but informal process | COMMERCIAL | GOVERNMENT compliance sections |
| Commercial but heavily regulated industry | COMMERCIAL | GOVERNMENT data security section |
| Partnership with government entity | PARTNERSHIP | GOVERNMENT compliance sections |
| Complex SOW for new client | SIMPLE/SOW | COMMERCIAL "Why Us" section |

---

## Template Customization Rules

### ✅ You CAN
- Add sections relevant to your opportunity
- Reorder sections within a template
- Expand section depth for complex topics
- Add industry-specific terminology

### ❌ You CANNOT
- Skip the Executive Summary
- Omit pricing/investment information
- Leave out the call to action / next steps
- Remove compliance sections for government deals
